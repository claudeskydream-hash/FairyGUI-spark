# FGUI 文本装饰实现记录：描边 / 阴影 / 下划线

> 记录星火(WasiCore)Label 上实现 FairyGUI 文本装饰的方案与踩坑，供后续维护参考。
> 涉及文件：`FGUI/Render/ISCEAdapter.cs`、`FGUI/Render/SCEAdapter.cs`、`FGUI/UI/FGUITextField.cs`。

## 结论速查

| 特性 | 引擎原生支持 | 实现方式 | 运行时开销 |
|------|:---:|------|------|
| 粗体 Bold / 斜体 Italic | ✅ | `Label.Bold/Italic` | 零 |
| **描边 Stroke** | ✅ | `Label.StrokeSize` + `Label.StrokeColor` | 零 |
| **阴影 Shadow** | ✅ | `Label.ShadowOffset` + `Label.ShadowColor` | 零 |
| 行间距 Leading | ✅ | `Label.LineSpacingRatio`（比率制，需换算，未接） | 零 |
| 字间距 LetterSpacing | ✅ | `Label.TextLetterSpacing`（未接） | 零 |
| **下划线 Underline** | ❌(无原生字段) | **主**：`<u>` 内联标记；**备**：覆盖 Canvas 手绘 | 主≈零 / 备：小 |
| **UBB 内联富文本** | 部分 | UBB→引擎内联标记透传（见第三节） | ≈零 |
| 删除线 Strikethrough | ❌ | 未实现（可试 `<s>` 标记；或备用覆盖层画中线） | — |

> 引擎 Label 文本渲染可解析部分内联标记，**实测**：`<u>` `<b>` `<i>` `<color=#RRGGBB>` ✅ 生效；`<size=..>` ❌ 不解析（会吐原文）。

星火原生 `Label`（`GameUI.Control.Primitive.Label` / 数据模板 `GameDataControlLabel` / schema 三处一致）**没有下划线、删除线**字段。

## 一、描边 / 阴影（原生，零开销）

引擎 `Label` 直接有属性，赋值即可，由引擎在文字栅格化时一并产出，无额外控件/绘制/分配。频繁 `SetText` 时这些属性不被触碰，改字成本不变。

- 接口：`ISCEAdapter.SetTextStroke(control, color, size)` / `SetTextShadow(control, color, offset)`
- 实现（`SCEAdapter`）：
  ```csharp
  label.StrokeSize = size; label.StrokeColor = color;
  label.ShadowColor = color; label.ShadowOffset = new Vector2(offset.X, offset.Y);
  ```
- 接线（`FGUITextField.UpdateDisplay`）：从二进制解析 `_strokeColor/_strokeSize`、`_shadowColor/_shadowOffset` 后直接调用；无值时显式清零，避免控件复用残留。
- ⚠️ 解析坑：原代码 `Setup_BeforeAdd` 读了描边 color 后把 size（`ReadFloat()`）丢弃了，需改成 `_strokeSize = (int)buffer.ReadFloat();`。

## 二、下划线

### 主方案（当前启用）：`<u>` 内联标记
引擎 Label 的文本渲染**能解析 `<u>下划线</u>` 内联标记**。做法：`FGUITextField.UpdateDisplay` 里 `_underline` 为真时把显示文本包成 `<u>{displayText}</u>` 再 `SetText`。
- 优点：**原生、贴合文字宽度、支持对齐/多行、零额外控件与开销**。
- 已知局限：线画在**文字 z 层之后**、纵向偏高，**中文/CJK 字形实心到底会把线盖住**，只在 ABC/数字空隙清晰露出；该位置/层级由引擎决定、外部不可调。若不可接受，切到下面的备用方案。
- 顺带可试：`<b>` 粗体、`<color=#RRGGBB>`、`<s>` 删除线等标记是否生效。

### 备用方案（已实现、当前不启用）：给 Label 叠一个兄弟 Canvas，立即模式画线
> 切换方式：把上面 `<u>` 包裹去掉，`FGUITextField` 里 `adapter.SetTextUnderline(...)` 改回传 `_underline`。
> 优点是线画在**文字之上、位置可控**（中文不被盖）；短板是量不到文字宽、只能整宽线（对 autoSize 文本域正确）。

核心代码在 `SCEAdapter`：`SetTextUnderline` / `SyncUnderlineOverlay` / `DrawUnderline` + `UnderlineState`，并在 `AddChild`/`SetPosition`/`SetSize` 末尾对有覆盖层的 Label 调 `SyncUnderlineOverlay`。

关键点与踩坑（都是实测踩出来的）：

1. **Label 不渲染子控件** —— 把 Canvas 作为 Label 的子级（`label.Add(canvas)`）时 `childCount=1` 但 Canvas 的 `OnRender` **永不触发**。
   → 必须把覆盖 Canvas 作为**兄弟**挂到 Label 的**父容器（Panel）**下（`parent.Add(overlay)`），并对齐 Label 的位置/尺寸、`ZIndex(30000)` 压在文字之上。父级从 `_attachedParentByChild` 取，位置从 `_controlPositions` 取。

2. **透明自绘 Canvas 的 `OnRender` 不触发** —— 现有能出图的 Canvas（椭圆/图片）是设了 image/fill 才被绘制；纯靠 OnRender 自绘的透明 Canvas 被引擎当空跳过。
   → 改用**立即模式**：挂好后直接调 `canvas.DrawLine(...)`（Canvas 文档主用法就是"创建后直接画"，绘制会持久保留）。

3. **`MeasureText` 在 `OnRender` 外返回不可靠** —— 立即模式下量文字宽度返回 0（引擎文档明确：OnRender 外前进宽度不准）。
   → 回退成**整宽线**（0 → 文本域宽）。对 **autoSize 文本域**（域宽≈文字宽）即为正确的文字下划线；固定宽且文字没占满（尤其居中/右对齐）时线会偏长，这是当前已知局限。

4. **避免每帧累积 / 刷屏** —— 该屏若每帧调 `SetPosition`，`SyncUnderlineOverlay`→`DrawUnderline` 会每帧触发（60fps）。
   → `DrawUnderline` 内加**变化守卫**（文字/字号/尺寸未变则不重画，立即模式绘制会持久保留）；重画前先 `canvas.ResetState()` 清掉上一条线，杜绝显示列表累积。

5. **纵向位置**：单行，按 `VerticalContentAlignment` 取所在行，`y = yTop + fontSize * 0.95`；粗细 `max(1, fontSize/16)`；颜色取 `label.TextColor`。系数可按字体基线微调。

### 关键 API
- `Canvas.DrawLine(x1,y1,x2,y2)`；线色 `StrokePaint`（Color 可隐式转），线宽 `StrokeWidth`。
- `Canvas.ResetState()`：清除已绘制内容 + 重置变换。
- `Canvas.MeasureText(x,y,text) → (advance, bounds)`：**仅在 OnRender 内返回可靠**。
- 加子控件 `parent.Add(control)`；`control.Position/Size/ZIndex/RemoveFromParent`。

## 三、UBB 内联富文本（已打通）

FairyGUI 文本域勾了「UBB Enabled」后可写 `[u]/[b]/[i]/[color=..]/[size=..]` 等标记。**原 port 把 UBB 全剥成纯文本（`UBBParser` 所有 handler 返回 ""、`FGUITextField` 调 `ToPlainText`），富文本全丢**。

现改为 **UBB → 引擎内联标记透传**：
1. `UBBParser` 的 handler 改成输出引擎标记：`[u]→<u>`、`[b]→<b>`、`[i]→<i>`、`[color=X]→<color=X>`；引擎不支持的（`size/url/img/sup/sub/font`）继续剥掉（仅去标签、保留文字，避免露出 `<tag>` 原文）。
2. `UBBParser.ParseHTML` 原本会丢弃 `<...>`，改为 `!remove` 时**透传**标签给引擎。
3. `FGUITextField` 的 `_ubbEnabled` 分支：`displayText = UBBParser.Instance.Parse(displayText)`（不再 `ToPlainText`）。

**实测引擎标记支持**（用 `[u][b][i][color=#FF0000][size=40]` 测）：

| 标记 | 引擎 | 处理 |
|---|:---:|---|
| `<u>` 下划线 | ✅ | 透传 |
| `<b>` 粗体 | ✅ | 透传 |
| `<i>` 斜体 | ✅ | 透传 |
| `<color=#RRGGBB>` | ✅ | 透传 |
| `<size=..>` / `<font size=..>` 字号 | ❌ 两种写法都吐原文 | 剥离（字号只能用字段级 FontSize，做不到同行多字号） |
| `url/img/sup/sub/font` | — | 剥离 |

> 后续若发现引擎新支持某标记，把 `UBBParser` 里对应 handler 从返回 "" 改成输出 `<tag>` 即可。

## 四、删除线（未实现，备注）
同下划线覆盖层，线画在中线（`y ≈ yTop + fontSize*0.5`）即可。但 **FairyGUI 标准编辑器/二进制没有删除线字段**，需自定义触发方式（自定义 UBB `[s]` 标签，或开 C# API）。

## 备选：若以后想让线精确贴合文字宽度
当前整宽线的唯一短板是非 autoSize + 居中/右对齐场景。要精确贴合需拿到真实文字宽度，可行方向：
- 想办法让覆盖 Canvas 的 `OnRender` 真正触发（届时 `MeasureText` 可靠），或
- 用引擎其它文字量测通道（若有 OnRender 外可靠的量测 API）。
在此之前，autoSize 文本域用整宽线已足够。
